Завдання 1: Виконати теоретичні розрахунки асимптотичної складності алгоритмів сортування методами вставляння та бульбашки.¶

Обидва алгоритми (сортування вставлянням та бульбашкою) базуються на вкладених циклах, що визначає їхню часову складність.

Сортування бульбашкою (Bubble Sort)

Найгірший та середній випадки: Масив відсортований у зворотному порядку. Кількість порівнянь: $\frac{n(n-1)}{2}$. $$T(n) = O(n^2)$$ Найкращий випадок: Масив уже відсортований. $$T(n) = O(n)$$ Сортування вставлянням (Insertion Sort)

Найгірший та середній випадки: Елементи знаходяться далеко від своїх позицій. $$T(n) = O(n^2)$$ Найкращий випадок: Масив уже відсортований (лише внутрішня перевірка без зсувів). $$T(n) = O(n)$$

import time
import random
import matplotlib.pyplot as plt

# Алгоритм сортування бульбашкою
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break
    return arr

Контрольні питання:¶

  1. Що таке O-нотація і чим вона відрізняється від Ω -нотації?

О-нотація: гарантує, що алгоритм не буде працювати повільніше за цей ліміт. Це "найгірший випадок".

Ω -нотація: гарантує, що алгоритм не буде працювати швидше за цей ліміт. Це "найкращий випадок".

  1. Яку часову складність має алгоритм сортування методом вставляння за найсприятливіших умов, коли масив відсортовано? Відповідь обґрунтувати.

Складність $O(n)$. Оскільки найкращий випадок - це відсортований масив, то ми просто проходимо по масиву один раз зліва направо.

  1. Надати визначення ефективного алгоритму.

Це алгоритм, час роботи якого зростає помірно при збільшенні вхідних даних, і який раціонально використовує пам’ять.

  1. Пояснити поняття головного параметра (розміру) завдання.

Це $n$ — розмір вхідних даних (кількість елементів у списку, кількість символів у файлі тощо). Саме від цього числа залежить, скільки операцій зробить процесор.

  1. Функція часової складності має вигляд: F(N) = N^3 + 7N^2 − 14N. Як записати асимптотичну складність у O-нотації?

Складність $O(n)$. Залишаємо тільки найстарший степінь, бо при величезних $N$ менші доданки та коефіцієнти майже не впливають на швидкість.
